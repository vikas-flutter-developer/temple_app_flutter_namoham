📱 Registration Flow (2-Step Process)

User Registration 

1. POST /auth/send-registration-otp
   {
     "phoneNumber": "+919876543210",
     "email": "test@example.com",
     "userType": "user"
   }
   → Check phone for OTP

2. POST /auth/registerUser
   {
     "fullName": "Test User",
     "email": "test@example.com",
     "phoneNumber": "+919876543210",
     "password": "Password123",
     "otp": "123456"  ← Enter OTP from SMS
   }
   → Account created! ✅

🔑 Password Reset Flow (2-Step Process) 

1. POST /auth/forgot-password
   {
     "email": "test@example.com",
     "userType": "user"
   }
   → Check phone for OTP

2. POST /auth/reset-password
   {
     "email": "test@example.com",
     "userType": "user",
     "phoneNumber": "+919876543210",
     "otp": "123456",  ← Enter OTP from SMS
     "newPassword": "NewPassword@123"
   }
   → Password changed! ✅

3. (Optional) POST /auth/resend-reset-otp
   If OTP expired, request a new one

🔐 Login 

POST /auth/login
{
  "email": "test@example.com",
  "password": "Password123",
  "userType": "user"  ← or "temple" or "creator"
}
→ Returns token for authenticated requests

📋 Common Response Patterns

✅ Success - OTP Sent 

{
  "message": "OTP sent successfully",
  "sessionId": "abc123..."
}

Action: Check your phone for SMS, then proceed to step 2

✅ Success - Registration Complete 

{
  "message": "User registered successfully",
  "user": { ... },
  "refreshToken": "..."
}

Action: Save the token, you're registered!

✅ Success - Password Reset 

{
  "message": "Password reset successfully. Please login with your new password.",
  "success": true
}

Action: Login with your new password

❌ Error - Invalid OTP 

{
  "message": "Invalid or expired OTP"
}

Action: Request a new OTP (use resend endpoint)

❌ Error - User Exists 

{
  "message": "User with this email or phone number already exists"
}

Action: Use different email/phone or login instead

🎯 Quick Testing Checklist

Before Testing

  • Update phone number to your actual number
  • Update email to your actual email
  • Server is running (npm run dev)
  • 2Factor API key is configured

User Registration Test

  • Send registration OTP
  • Receive SMS within 30 seconds
  • Enter OTP in registration request
  • User account created
  • Token received and saved

Password Reset Test

  • Request password reset OTP
  • Receive SMS within 30 seconds
  • Enter OTP in reset request
  • Password changed successfully
  • Login with new password works

Error Handling Test

  • Try expired OTP (wait 10 mins)
  • Try wrong OTP
  • Try registering duplicate user
  • Verify error messages are clear

🔧 Troubleshooting Quick Fixes

Problem Quick Fix
No SMS received Check phone number format: +91XXXXXXXXXX
"Invalid OTP" Make sure you entered all 6 digits correctly
"OTP expired" Use resend endpoint to get new OTP
"Session not found" Run step 1 (send OTP) first
"User exists" Use different email/phone or login
"Server error" Check if npm run dev is running
"2Factor error" Verify API key in .env file

⏱️ Important Timings

  • OTP Validity: 10 minutes (600 seconds)
  • SMS Delivery: Usually within 5-30 seconds
  • Session Timeout: Expires with OTP after 10 minutes
